# Доказовість

Ця сторінка описує, що фіксується, чого не заявляється, та як відстежуються зміни.

---

## Що ми фіксуємо

- **Історія версій** — Усі зміни відстежуються в Git з хешами комітів.
- **Журнал змін** — Зрозумілий людині підсумок релізів та модифікацій.
- **Артефакти рішень** — Структурні рішення задокументовані у файлах репозиторію.
- **Часові мітки** — Кожен коміт включає метадані дати та авторства.

---

## Чого ми НЕ заявляємо

- **Немає гарантії придатності** — Контент надається "як є" без гарантій.
- **Немає формальної заяви про відповідність** — Цей сайт не видає сертифікатів відповідності.
- **Немає валідації третіми сторонами** — Контент не перевірявся зовнішніми органами.
- **Немає операційної застосовності** — Документація не є керівництвом з впровадження.

---

## Як відстежуються оновлення

Усі зміни керуються через систему контролю версій Git:

- **Коміти** — Кожна зміна записується з унікальним ідентифікатором.
- **Теги** — Релізи можуть бути позначені тегами версій для довідки.
- **Відкат** — Попередні версії можуть бути відновлені з історії Git.
- **Публічний журнал** — Див. [Журнал змін](/ua/change-log/) для підсумку.
